# [OS] 스케줄러(Scheduler)의 기본 개념 정리

## 1. 스케줄러란?
한정된 시스템 자원(CPU 등)을 여러 프로세스나 스레드에게 효율적으로 배분하기 위해 실행 순서를 결정하는 **운영체제의 핵심 관리 도구**입니다.

### 핵심 목적
- **자원 활용 극대화:** CPU가 쉬지 않고 일하게 함 (CPU Utilization)
- **공정성:** 모든 프로세스에 공평하게 자원을 배분
- **응답 시간 단축:** 대화형 시스템에서 사용자에게 빠른 반응 제공
- **처리량(Throughput) 증대:** 단위 시간당 완료되는 작업 수 증가

---

## 2. 프로세스 상태 모델
스케줄러는 프로세스의 상태를 변화시키며 관리합니다.

1.  **Ready (준비):** CPU를 할당받기 위해 메모리 내 큐(Queue)에서 대기하는 상태
2.  **Running (실행):** CPU를 점유하여 실제로 명령어를 수행 중인 상태
3.  **Waiting/Blocked (대기):** I/O 작업이나 이벤트 완료를 기다리며 일시 정지된 상태

---

## 3. 스케줄러의 종류 (계층별 구분)

| 구분 | 이름 | 역할 | 실행 주기 |
| :--- | :--- | :--- | :--- |
| **단기 스케줄러** | CPU Scheduler | 준비 큐의 프로세스 중 어떤 프로세스에 CPU를 할당할지 결정 | 매우 잦음 (ms 단위) |
| **중기 스케줄러** | Swapper | 메모리 부족 시 프로세스를 디스크로 내보내(Swap-out) 빈 공간 확보 | 필요 시 |
| **장기 스케줄러** | Job Scheduler | 디스크에 있는 프로그램 중 어떤 것을 메모리에 올려 프로세스로 만들지 결정 | 드묾 (최근 OS는 생략하기도 함) |

---

## 4. 스케줄링 방식의 분류

### 1) 비선점형 (Non-preemptive)
- 한 프로세스가 CPU를 잡으면 작업이 끝날 때까지 CPU를 뺏을 수 없음.
- **장점:** 문맥 교환(Context Switch) 부하가 적음.
- **예:** FCFS(선착순), SJF(최단 작업 우선)

### 2) 선점형 (Preemptive)
- 운영체제가 강제로 CPU 사용권을 뺏어 다른 프로세스에게 줄 수 있음.
- **장점:** 빠른 응답성이 필요한 시분할 시스템에 적합.
- **예:** Round Robin(라운드 로빈), SRTF(최단 잔여시간 우선)

---

## 5. 대표적인 스케줄링 알고리즘
- **FCFS (First-Come, First-Served):** 먼저 도착한 순서대로 처리 (선착순).
- **SJF (Shortest Job First):** 수행 시간이 짧은 작업을 먼저 처리.
- **Round Robin (RR):** 각 프로세스에 동일한 시간(Time Quantum)을 할당하고 순환하며 처리.
- **Priority Scheduling:** 우선순위가 높은 프로세스부터 처리.
- **Multilevel Queue:** 프로세스 성격에 따라 여러 개의 큐를 사용.

---

## 6. 주요 용어 요약
- **Context Switch (문맥 교환):** CPU가 한 프로세스에서 다른 프로세스로 전환될 때, 현재 상태를 저장하고 새로운 상태를 로드하는 과정.
- **Time Quantum (타임 슬라이스):** 프로세스가 CPU를 독점할 수 있도록 허용된 짧은 시간.
- **Starvation (기아 상태):** 우선순위가 낮아 오랫동안 CPU를 할당받지 못하는 현상.
