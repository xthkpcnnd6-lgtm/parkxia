# 🗄️ 데이터베이스(Database) 기초 핵심 요약

> 데이터베이스의 핵심 개념부터 설계, SQL의 기본까지 한눈에 볼 수 있도록 정리한 문서입니다.

---

## 1. 데이터베이스의 기본 개념

### 📌 정의
- **데이터베이스(DB):** 체계적으로 관리되는 데이터의 집합.
- **DBMS (Database Management System):** 데이터베이스를 관리하고 사용자의 요구에 따라 데이터를 제어하는 소프트웨어 (예: MySQL, Oracle, PostgreSQL).

### 📌 특징
1. **실시간 접근성(Real-time Accessibility):** 사용자의 질의에 즉시 응답.
2. **계속적인 변화(Continuous Evolution):** 삽입, 삭제, 수정을 통해 최신 상태 유지.
3. **동시 공유(Concurrent Sharing):** 여러 사용자가 동시에 이용 가능.
4. **내용에 의한 참조(Content Reference):** 주소가 아닌 데이터 값에 따라 참조.

## 2. 데이터 모델링 (설계 단계)

데이터베이스를 구축하기 전, 현실 세계의 데이터를 구조화하는 과정입니다.

| 단계 | 주요 활동 | 비고 |
| :--- | :--- | :--- |
| **개념적 설계** | 정보 구조 결정, **E-R 다이어그램** 작성 | 사용자 중심 |
| **논리적 설계** | DBMS에 맞는 논리적 구조(테이블) 생성 | 정규화 수행 |
| **물리적 설계** | 저장 레코드 양식, 물리적 저장 장치 결정 | 성능 최적화 |

---

## 3. 관계형 데이터베이스(RDB) 핵심 용어

관계형 데이터베이스는 데이터를 **표(Table)** 형태로 관리합니다.

- **엔티티(Entity):** 객체 또는 사물 (예: 학생, 교수, 강의).
- **속성(Attribute):** 엔티티의 특성, 표의 **열(Column)**.
- **튜플(Tuple):** 엔티티의 인스턴스, 표의 **행(Row)**.
- **도메인(Domain):** 한 속성이 가질 수 있는 원자값의 범위.

### 🔑 키(Key)의 종류
* **기본키 (Primary Key):** 튜플을 유일하게 식별할 수 있는 고유한 키 (중복 불가, NULL 불가).
* **외래키 (Foreign Key):** 다른 테이블의 기본키를 참조하는 키 (테이블 간 관계 연결).
* **후보키 (Candidate Key):** 기본키가 될 수 있는 자격을 갖춘 속성들.

## 4. 정규화 (Normalization)

데이터의 중복을 최소화하고 이상 현상(Anomaly)을 방지하기 위해 테이블을 분해하는 과정입니다.

1.  **제1정규형(1NF):** 모든 속성 값이 **원자값**을 가져야 함.
2.  **제2정규형(2NF):** 부분 함수적 종속 제거 (기본키가 복합키일 때 발생).
3.  **제3정규형(3NF):** 이행적 함수적 종속 제거 (A->B, B->C 이면 A->C 관계인 것 제거).

## 5. SQL (Structured Query Language) 분류

| 분류 | 설명 | 주요 명령어 |
| :--- | :--- | :--- |
| **DDL** (정의어) | 데이터베이스 구조를 생성/수정/삭제 | `CREATE`, `ALTER`, `DROP`, `TRUNCATE` |
| **DML** (조작어) | 데이터를 조회/삽입/수정/삭제 | `SELECT`, `INSERT`, `UPDATE`, `DELETE` |
| **DCL** (제어어) | 데이터 접근 권한 부여/취소 | `GRANT`, `REVOKE` |
| **TCL** (트랜잭션) | 트랜잭션 완료/복구 | `COMMIT`, `ROLLBACK` |

---

## 6. 트랜잭션 (Transaction) 및 ACID

데이터베이스의 상태를 변화시키는 하나의 논리적 작업 단위입니다.

- **A (Atomicity, 원자성):** 모두 반영되거나, 전혀 반영되지 않아야 함 (All or Nothing).
- **C (Consistency, 일관성):** 실행 완료 후 데이터베이스는 항상 일관된 상태여야 함.
- **I (Isolation, 격리성):** 둘 이상의 트랜잭션이 동시 실행될 때 서로 간섭할 수 없음.
- **D (Durability, 영속성):** 성공한 트랜잭션의 결과는 영구적으로 저장됨.

---

## 7. 인덱스 (Index)

- **목적:** 데이터 검색 속도를 향상시키기 위해 사용합니다.
- **장점:** 빠른 조회가 가능함.
- **단점:** `INSERT`, `UPDATE`, `DELETE` 시 성능 저하가 발생할 수 있으며 추가 저장 공간이 필요함.
